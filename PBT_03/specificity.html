Phần 2: CSS Rules Applied

Thứ tự CSS Rule Specificity Màu (Dự đoán) Thắng?
1 p { color: red; } (0, 0, 1) Red ❌
2 .text { color: blue; } (0, 1, 0) Blue ❌
3 .highlight { color: green; } (0, 1, 0) Green ❌
4 p.text { color: purple; } (0, 1, 1) Purple ❌
5 p.highlight { color: orange; } (0, 1, 1) Orange ❌
6 p.text.highlight { color: brown; } (0, 2, 0) Brown ❌
7 #demo { color: pink; } (1, 0, 0) Pink ✅ THẮNG!
8 p#demo { color: cyan; } (1, 0, 1) Cyan ❌
9 p.text#demo { color: teal; } (1, 1, 1) Teal ❌
10 .text.highlight#demo { color: gold; } (1, 2, 0) Gold ❌

Giải thích

Rule 7 (`#demo { color: pink; }`) THẮNG vì:

  • ID selector có specificity cao nhất: (1, 0, 0) = 100 điểm
  • Các rule khác có specificity thấp hơn:
    • Rules 1-3: (0, 0, 1) và (0, 1, 0) = 1 hoặc 10 điểm
    • Rules 4-6: (0, 1, 1) hoặc (0, 2, 0) = 11 hoặc 20 điểm
    • Rule 8: (1, 0, 1) = 101 điểm, nhưng xuất hiện sau Rule 7
    • Rules 9-10: (1, 1, 1) hoặc (1, 2, 0) = 111 hoặc 120 điểm, nhưng xuất hiện sau
  • Nguyên tắc: Specificity cao hơn → chiến thắng. Nếu bằng nhau → rule xuất hiện sau thắng (cascade)

Câu hỏi

1. Nếu chuyển đổi thứ tự CSS (Rule 7 ở cuối thay vì ở giữa), kết quả có thay đổi không?

Trả lời: KHÔNG thay đổi! Vì Rule 7 có ID selector (specificity 100), cao hơn tất cả các rule khác. Vị trí không quan trọng khi specificity khác nhau.

2. Nếu xóa Rule 7 (#demo), rule nào sẽ thắng?

Trả lời: Rule 6 (`p.text.highlight { color: brown; }`) hoặc Rules 9-10 nếu chúng xuất hiện sau. Cùng specificity thì rule xuất hiện sau thắng.
